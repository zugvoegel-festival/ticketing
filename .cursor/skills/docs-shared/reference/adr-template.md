# ADR template

Use ADRs for significant decisions that affect future changes (new architecture constraint, major dependency, security model, etc.).

```markdown
# ADR NNNN: <short decision name>

Date: YYYY-MM-DD

## Context

What problem are we solving? What constraints matter?

## Decision

What did we decide, in one or two sentences?

## Options considered

- Option A: ...
- Option B: ...

## Consequences

- Pros: ...
- Cons / tradeoffs: ...

## Follow-ups

- [ ] Tasks to implement or migrate
```
